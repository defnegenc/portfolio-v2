# Hercules — Project Context

**Type**: Hackathon entry
**Category**: Design (portfolio)
**Tags**: Hackathon Entry, UX Design, AI Agent
**Accent color**: #FF6B35 (orange)

## What Hercules Is

An AI agent for mobility tracking and understanding. Addresses the "healthspan-lifespan gap" — life expectancy has grown by 30 years since the mid-twentieth century but healthspan hasn't followed. Hercules tackles loss of mobility specifically.

Users interact via voice and image: point to or describe where they're experiencing pain → Hercules asks follow-up questions and logs symptoms. Two modes: (1) follow up on a previous pain/discomfort, (2) report a new one.

## Defne's Role

- **Product scoping** — defined the problem and feature set
- **UX journey** — end-to-end user flow design
- **UI design** — Figma mockups for pain logging flow and dashboard, ready for development and testing

## Key Learning

Making UX decisions that not only address immediate health concerns but also promote long-term well-being. Prioritizing features and design elements that encourage proactive health monitoring and early intervention (vs. reactive symptom tracking).

## Portfolio Framing

Hercules demonstrates:
- Product sense (scoped a real health problem from a population-level insight: healthspan-lifespan gap)
- Health UX (sensitive domain — pain logging requires clarity, trust, no friction)
- AI product design (voice + image as primary input modalities, no typing required)
- Hackathon speed and execution (TreeHacks 2024)
- Proactive vs reactive design thinking: prioritized patterns and early intervention over symptom logging
